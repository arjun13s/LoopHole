# `self_improve/fixtures/` — golden acceptance oracle

`improvement_cases.jsonl` — 16 rows, regenerated by `generate_golden.py` from the **real** reward
path (`eval_harness.build_eval_record`) + signal producer (`diagnostics.build_sidecar_record`).

Each row:

```json
{"id": "...", "note": "...", "eval_result": {...}, "sidecar": {...}, "expect": {...} | null}
```

- `eval_result` validates against the frozen `schemas/eval_result.json`.
- `sidecar` validates against `schemas/verdict_sidecar.json` (and its verdict core against the
  frozen `schemas/verdict.json`).
- **`expect` is a PARTIAL assertion oracle, not a full `ImprovementRecord`.** It deliberately
  asserts only `{bucket, fix_type, contributing_factors, confidence, severity, suggested_alias}`
  and omits the schema-required `evidence`/`diagnosis`/`suggested_action` (those are free-text the
  analyzer constructs). **Do not validate `expect` against `schemas/improvement_record.json`** — it
  is not meant to conform. The analyzer's *constructed* record is what must satisfy that schema.
- `expect == null` means `classify` must return `None` (a healthy run → no record).

Codex's test: for each row, `classify(row.eval_result, row.sidecar)` matches `row.expect`
(compare `contributing_factors` order-insensitively). Coverage: all 10 buckets + the None-path.
